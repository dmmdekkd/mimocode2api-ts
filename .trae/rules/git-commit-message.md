# Git 提交信息生成规则

## 基本格式
- 使用中文编写提交信息
- 遵循 Conventional Commits 规范
- 格式: `<类型>(<可选作用域>): <描述>`

## 类型定义
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具/依赖更新

## 编写要求
1. 标题简洁明了，不超过 50 个字符
2. 描述使用动词开头，如"添加"、"修复"、"更新"等
3. 需要时添加详细正文说明变更原因和影响
4. 关联相关 Issue 或 PR 编号
5. 避免使用模糊的描述，如"修改了一些代码"

## 示例
- `feat(auth): 添加用户登录功能`
- `fix(api): 修复空指针异常`
- `docs(readme): 更新安装说明`
